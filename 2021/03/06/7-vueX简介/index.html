7.vueX详细篇

vueX简介

vuex实现组件间数据共享的方案，可以方便组件之间的传值

vuex好处：

1.能够在vuex中几种管理共享的数据。易于开发和后期维护

2.能搞高效的实现组件之间的共享数据，提高开发效率

3.存储在vuex中的数据都是响应式的，实时保持数据与页面的同步

注意：只有共享的数据可以存储到vuex中，组件内的独有数据，存在自己的data中

使用

  • 安装依赖

    • ```
      npm install vuex –save

  • 创建store对象

    • new Vuex.store({
    // state中存放的时全局共享的数据
    state:{
      count:0
    }
})

new Vue({
el: '#app',
components: { App,vuex},
template: '<App/>'
})

State 提供了唯一的公共数据源，所有的共享数据就都要放在state内

this.$store.State.数据名称
// 例如
<p>当前count数据是：{{$store.state.count}}</p>

import { mapstate }  from 'vuex'

// 2. 将全局数据，映射为当前组件的计算属性
computed:{
 ...mapState(['count'])
}
// 使用
<p>{{ count }}<p>

this.$store.commit("add",step)  // Mutation中定义的方法   step 为步长 

methods:{
  ...mapMutations(['add','addN'])   // add,addn 分别为 Mutations中自定义的两种方法

state: {
    count: 0
},

mutations: {
adds(state,setp){
state.count += setp

},
// 用于处理异步任务
actions: {
addAsync(content，setp){
setTimeout(() => {
content.commit(“adds”,setp)
}, 1000);
}
},

// 页面中使用


<button @click=”asyncs”>async+1

methods:{

const store = new vuex.Store({
state: {
count: 0
},
// 只有mutations中定义的函数 才可以修改state中的数据
mutations: {
add(state,step){
state.count += step
},
},
// 用于处理异步任务
actions: {
addAsync(content,step){
setTimeout(() => {
content.commit(“add”,step)
}, 1000);
}
},
})

import { mapActions } from "vuex";

methods:{
…mapActions([‘addAsync’]),
asyncs(){
// 第一种方式

this.addAsync(3)
},
}

// 用于对state中的数据进行加工，形成新的数据， 类似于 vue中的计算属性

}
